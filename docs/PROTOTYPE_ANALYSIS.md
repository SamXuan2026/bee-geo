## 3. 业务链路

```text
基础资产建设
  -> 关键词库 / 知识库 / 素材库 / AI 人设 / 集成账号
GEO 分析
  -> 关键词输入 / 模拟问题 / AI 引用分析
AI 创作
  -> 文章类型 / 提示词 / 正文编辑 / 草稿保存
审核发布
  -> 提交审核 / 发送设置 / 定时发布 / 发布回执
```

## 4. 原型缺口

- 缺少总览仪表盘。
- 自动化发布没有独立发布中心。
- 缺少发布状态机、失败重试和人工介入规则。
- 授权凭据缺少加密、脱敏、过期提醒和审计设计。
- 权限模型较粗，需要区分内容、审核、发布和只读角色。

## 5. 产品化处理

- 将发布能力从 AI 创作尾部步骤抽象为一级模块。
- 将 AI 模型、发布平台、文件存储、凭据管理抽象为端口接口。
- 前端按模块组织 `pages / api / model / components`，后续局部变化不影响主体工程。
