# UI 设计规范

## 1. 视觉方向

参考用户提供的金融仪表盘图片，bee-geo 后台采用浅灰蓝、玻璃质感、柔和阴影和大号数据展示。

## 2. 页面结构

- 顶部导航：品牌、一级模块、业务场景标签、通知和用户头像。
- 左侧模块坞：按总览、洞察、内容、发布、资产、系统分组。
- 内容区：顶部大标题和弧形指标图，下方为卡片、表格和 AI 助手区域。

## 3. 色彩

| 用途 | 颜色 |
| --- | --- |
| 背景 | `#eaf3fa`、`#dceaf4` |
| 主文本 | `#0f1720` |
| 次级文本 | `#718092` |
| 主蓝 | `#278be8` |
| 成功 | `#19b37b` |
| 警告 | `#f6b73c` |
| 失败 | `#e34863` |

## 4. 组件规范

- 卡片统一使用半透明白底、1px 轻边框、柔和阴影。
- 圆角控制在 8px，保证企业后台的克制感。
- 表格保持高行距、轻分割线和明确操作按钮。
- 状态标签使用浅色底和高对比文字。
- AI 助手卡片可使用深色渐变，但不能影响主体信息阅读。

## 5. 可维护要求

- 全局样式集中在 `src/app/styles.css`。
- 业务页面不直接写复杂视觉样式，优先复用 `shared/ui.tsx`。
- 新增页面必须保持 `PageHeader + Panel + Toolbar + Table/Card` 结构。
- 不引入远程图片、远程字体和外网资源。
