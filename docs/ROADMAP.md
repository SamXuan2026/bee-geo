# 迭代路线图

## 第 0 迭代：设计基线与工程准备

- 完成文档、工程目录、前端原型、后端骨架。
- 验收：文档可指导 AI 执行，前端可演示，后端接口边界清晰。

## 第 1 迭代：前端管理后台原型

- 完成所有核心页面的静态和 mock 交互。
- 验收：无需后端即可演示完整主流程。

## 第 2 迭代：前端业务交互与状态管理

- 模块内补分页、筛选、弹窗、上传占位、状态流转。
- 验收：替换 API service 即可接入真实后端。
- 当前进度：核心按钮已补齐可见反馈；GEO、创作、发布、资产、关键词、知识库、人设、用户和集成设置已支持搜索、筛选、弹窗、创建、编辑、删除或状态流转。后端不可用时启用本地兜底，并通过浏览器本地存储保持刷新后数据不丢。
- 验证进度：已新增 `npm run page-smoke`，自动检查死按钮、受控表单、关键本地兜底和主链路标记。

## 第 3 迭代：后端基础工程与核心数据

- 实现基础 CRUD、权限占位、审计拦截器、文件上传。
- 验收：基础数据 API 可用。

## 第 4 迭代：GEO 分析与 AI 创作后端

- 实现 GEO 任务、结果转创作、草稿保存、审核流转。
- 验收：GEO 到创作闭环可用。
- 当前进度：GEO 任务、GEO 结果、AI 创作草稿已持久化；GEO 结果可转创作草稿；内容编辑保存、提交审核、审核通过和创建发布排期已接入。
- 前端兜底进度：无后端时，GEO 创建草稿会写入本地创作列表，跳转到 AI 创作页后可继续编辑、审核和排期。

## 第 5 迭代：自动化发布中心

- 实现发布任务、适配器、凭据服务、失败重试、回执。
- 验收：审核通过内容可排期并发布到自有站点或免费媒体。
- 当前进度：发布任务、发布账号、发布回执已持久化；前端发布中心已接入真实 API；到期任务调度器已接入；凭据已改为加密持久化。
- 前端兜底进度：无后端时，创建发布任务、创作排期、重试、撤回和回执查看均可写入浏览器本地存储，刷新页面后仍可见。

## 第 6 迭代：权限、审计与安全加固

- 实现角色权限、敏感操作二次确认、审计查询。
- 验收：无权限不可发布，关键操作可追溯。
- 当前进度：后端已有审计日志接口；前端已新增审计日志模块，支持按关键词和结果筛选、查看详情；本地兜底操作会追加审计记录，用于无后端演示追溯链路。
- 权限进度：前端已新增权限矩阵模块，可按角色和风险等级筛选权限点，查看授权角色、风险等级和后端保护状态。
- 后端进度：已新增 `/api/security/permissions` 权限矩阵接口，支持关键词、角色编码和风险等级筛选，前端优先读取真实接口并保留本地兜底。
- 测试进度：已新增 `scripts/backend-test.sh` 和权限矩阵 JUnit 测试；针对当前本地 JDK/沙箱环境固定 Mockito 使用子类 mock maker，避免 inline 自挂载失败。
- 审计增强：已新增 `/api/audit/logs/page` 分页接口，支持关键词、操作结果、起止日期、页码和每页条数；前端审计页已同步增加日期筛选和分页控件。
- 审计导出：已新增 `/api/audit/logs/export` CSV 导出接口；前端审计页支持按当前筛选条件导出并预览 CSV，后端不可用时使用本地审计数据兜底。

## 第 7 迭代：联调、部署与验收

- 完成前后端联调、部署文档、测试用例、内网部署验证。
- 验收：形成可继续迭代的 MVP。
- 当前进度：前端已按参考图调整为浅色拟态监控台风格，包含浅蓝背景、青绿色主色、左侧窄导航、拟态卡片、紧凑信息区、表格行卡片和状态标签；前端构建、页面冒烟检查、后端测试均已通过。
- 首页增强：总览页已补齐环境仪表、控制开关、内容温度/引用湿度、运营趋势、设备状态柱状图和告警分布，视觉更接近参考图的监控台首屏；已增加窄屏降级规则。
- 主链路增强：GEO 分析页已新增信号指标、引用覆盖环、来源占比、任务状态控制块，形成“监控面板 + 明细列表 + 引用卡片”的结构，原创建分析、筛选、查看、删除和转创作流程保留。
- 视觉收尾：AI 创作页已新增生成质量、审计覆盖、草稿/审核/排期状态、正文完成度环；发布中心已新增发布成功率、回执归档、发布健康环、账号有效率和风险任务占比。总览、GEO、AI 创作、发布中心已统一为浅色拟态监控台风格。
- 验证加固：`npm run page-smoke` 已增加首页、GEO、AI 创作、发布中心核心监控面板标记校验，避免后续迭代回退为普通表格页。
- 后端测试补强：已新增审计分页与 CSV 导出服务测试，覆盖分页参数边界、按创建时间倒序、CSV 中文表头、失败结果输出和双引号转义。当前后端测试共 4 个用例通过。
- 发布测试补强：已新增发布任务服务测试，覆盖创建任务时标题正文裁剪、重试上限归一化、成功发布回执保存、发布成功状态流转、超过重试上限转人工介入和审计事件写入。当前后端测试共 7 个用例通过。
- 创作测试补强：已新增 AI 创作服务测试，覆盖创建草稿、提交审核、审核通过、审核通过后创建发布排期、未审核禁止排期和审计事件写入。
- GEO 测试补强：已新增 GEO 服务测试，覆盖关键词裁剪、AI 问题生成、GEO 结果保存、空结果禁止转草稿、转创作正文引用问题和审计事件写入。当前后端测试共 13 个用例通过。
- 权限加固：已为关键词、知识库、素材、人设的新增、编辑、删除接口，以及 AI 人设生成接口补充 `SUPER_ADMIN`、`CONTENT_ADMIN` 角色限制；已新增反射测试防止配置资产类写接口漏标权限。当前后端测试共 14 个用例通过。
- 前端权限闭环：已新增当前角色切换器和前端权限工具，按角色禁用 GEO 创建/转草稿、创作保存/审核/排期、发布创建/重试/撤回、集成授权、配置资产写操作和用户管理写操作；安全模块导航对非超级管理员禁用。页面冒烟已加入权限标记校验。
- 凭据测试补强：已新增凭据加密服务和 JPA 凭据适配器测试，覆盖 AES-GCM 加解密、非法密文、错误密钥、加密保存、脱敏读取、发布解密、平台不匹配和凭据过期。当前后端测试共 21 个用例通过。
- 配置资产测试补强：已新增关键词、知识库、素材服务测试，覆盖创建裁剪、空值归一化、默认启用、更新保留启用状态、删除和审计事件。当前后端测试共 27 个用例通过。
- 用户与人设测试补强：已新增用户服务和 AI 人设服务测试，覆盖用户账号唯一性、角色编码到角色名称映射、默认启用状态、用户更新删除审计、人设空值归一化、默认启用和人设更新删除审计。当前后端测试共 32 个用例通过。
- 部署检查：已为 Compose 补齐 PostgreSQL 服务、生产数据库连接、凭据加密密钥强制注入、关闭演示凭据种子和操作人请求头要求；新增 `scripts/deploy-check.sh`，可检查前端产物、Nginx 代理、安全响应头、核心服务、生产环境变量和 PostgreSQL 初始化表。
- 主链路回归：已新增 `npm run flow-smoke`，零依赖检查 `GEO 分析 -> AI 创作 -> 审核 -> 发布排期 -> 发布中心 -> 回执 -> 审计/权限` 的关键入口、API 契约、本地兜底和权限标记。后续可升级为 Playwright 浏览器点击测试。
- 凭据审计补强：发布账号重新授权保存凭据、凭据过期标记已写入 `integration` 模块审计事件，测试同步校验 `saveCredential` 和 `expireCredential`。
- 运行态 API 冒烟：已新增 `scripts/api-smoke.mjs` 和 `scripts/api-smoke.sh`，对运行中的后端执行只读检查，覆盖健康、基础数据、GEO、创作、发布、集成、权限矩阵和审计导出。
- 前端交互契约：已新增 `npm run interaction-smoke`，检查核心按钮、弹窗、表单、分页、审计导出、授权和发布链路是否绑定到明确状态变更或接口动作；`PageHeader` 类型已加固，传入操作文案时必须传入点击函数。
- 浏览器点击冒烟：已新增 `npm run browser-click-smoke`，默认通过 `file://` 打开 `dist` 构建产物并启动 Chrome Headless，通过 DevTools 协议真实点击导航、GEO、创作、审核、排期、发布、回执、授权、审计导出和角色切换并校验反馈；如需静态服务模式可设置 `BEE_GEO_BROWSER_MODE=server`。
- 后端 API 集成测试：已新增 `BackendApiIntegrationTest`，通过 MockMvc 覆盖健康、基础数据、GEO、创作、发布、集成、权限、审计导出、权限拒绝、凭据脱敏和 GEO 到发布排期主链路，不依赖本机端口。
- 运行态验收脚本契约修复：`scripts/api-smoke.mjs` 已同步后端当前审计导出契约，CSV 类型断言为 `text/csv;charset=utf-8`，表头断言为 `模块,动作,对象编号`。
- 本机运行态验收编排：已新增 `scripts/runtime-smoke.sh`，可在本机终端一键启动后端、等待健康检查、执行 API 冒烟、构建前端、执行浏览器点击冒烟并清理后端进程；支持跳过浏览器、跳过 API、使用已有后端、保留后端和调整等待时间。
- 当前验证：`npm run interaction-smoke`、`npm run flow-smoke`、`npm run page-smoke`、`npm run build`、`./scripts/backend-test.sh`、`bash scripts/deploy-check.sh`、`node --check scripts/api-smoke.mjs`、`node --check frontend/scripts/browser-click-smoke.mjs` 均已通过；后端测试当前 36 个用例通过。当前沙箱限制 Chrome Headless 执行和脚本内部子进程访问本机端口，`./scripts/api-smoke.sh` 与 `npm run browser-click-smoke` 需在本机终端或获得执行权限后复验。
