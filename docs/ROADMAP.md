# 迭代路线图

## 第 0 迭代：设计基线与工程准备

- 完成文档、工程目录、前端原型、后端骨架。
- 验收：文档可指导 AI 执行，前端可演示，后端接口边界清晰。

## 第 1 迭代：前端管理后台原型

- 完成所有核心页面的静态和 mock 交互。
- 验收：无需后端即可演示完整主流程。

## 第 2 迭代：前端业务交互与状态管理

- 模块内补分页、筛选、弹窗、上传占位、状态流转。
- 验收：替换 API service 即可接入真实后端。

## 第 3 迭代：后端基础工程与核心数据

- 实现基础 CRUD、权限占位、审计拦截器、文件上传。
- 验收：基础数据 API 可用。

## 第 4 迭代：GEO 分析与 AI 创作后端

- 实现 GEO 任务、结果转创作、草稿保存、审核流转。
- 验收：GEO 到创作闭环可用。
- 当前进度：GEO 任务、GEO 结果、AI 创作草稿已持久化；GEO 结果可转创作草稿；内容编辑保存、提交审核、审核通过和创建发布排期已接入。

## 第 5 迭代：自动化发布中心

- 实现发布任务、适配器、凭据服务、失败重试、回执。
- 验收：审核通过内容可排期并发布到自有站点或免费媒体。
- 当前进度：发布任务、发布账号、发布回执已持久化；前端发布中心已接入真实 API；到期任务调度器已接入；凭据已改为加密持久化。

## 第 6 迭代：权限、审计与安全加固

- 实现角色权限、敏感操作二次确认、审计查询。
- 验收：无权限不可发布，关键操作可追溯。

## 第 7 迭代：联调、部署与验收

- 完成前后端联调、部署文档、测试用例、内网部署验证。
- 验收：形成可继续迭代的 MVP。
